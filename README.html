# 🎯 Zomercompetitie Een open-source, mobielvriendelijke (PWA) FastAPI webapplicatie voor het beheren van een wekelijkse dartscompetitie. Gebouwd met een focus op gebruiksgemak voor de organisatie: snelle invoer van uitslagen vanaf een tablet, automatische poule-generatie, een ingebouwde update-checker én een volautomatisch TV-toernooischerm inclusief DartConnect-integratie. ## ✨ Features ### 📱 Organisatie & Beheer - **Slimme Aanwezigheid:** Spelers aanwezig melden via toggles (met onthouden van scroll-positie voor snelle invoer). - **Eerlijke Poules:** Automatische indeling met anti-herhaling op basis van historische ontmoetingen. - **Geavanceerde Knock-out:** Ondersteunt automatische 'vrijlotingen' (byes) bij afwijkende aantallen (zoals een 3-speler knock-out). - **Statistieken & Highlights:** Bulk-invoer voor uitslagen inclusief 180'ers, 100+ finishes en snelle legs (≤15 darters). - **Seizoensarchief:** Overall standen, legsaldo's en een compleet archief van voorgaande avonden. ### 📺 Live TV Toernooischerm (`/?tv=1`) - **Ultrawide Dark Mode:** Een prachtig volautomatisch toernooischerm dat perfect schaalt van 1080p TV's tot brede Ultrawide monitoren, gestyled in de kleuren van de app. - **DartConnect Integratie:** Voeg eenvoudig live DartConnect wedstrijd-URL's toe via het Admin-paneel. Deze worden in realtime naast het competitieschema op de TV getoond. - **Slimme Wedstrijd-logica:** Openstaande wedstrijden uit verschillende poules worden automatisch om-en-om "geritst", zodat het toernooi op alle banen soepel doorloopt. - **Auto-Refresh:** Het scherm draait volledig autonoom en ververst zichzelf elke 30 seconden. - **🏆 Huldigingsscherm:** Zodra een speelavond op `CLOSED` wordt gezet, transformeert de TV automatisch naar een uitslagenscherm met de winnaar van de avond én de actuele overall standenlijst. ### ⚙️ Systeem - **Update Checker:** Ingebouwde notificatie in het admin-paneel zodra er een nieuwe release op GitHub staat. ## 🛠 Tech stack - **Backend:** Python 3.10+, FastAPI, SQLAlchemy, SQLite - **Frontend:** Jinja2 Templates, Vanilla JS/CSS (Responsive & PWA-ready) - **Deployment:** Nginx + systemd (via meegeleverd bash script) --- ## Installatie op Ubuntu server Gebruik het install script vanuit de repo root: ```bash bash ./scripts/install_ubuntu.sh ### Belangrijkste omgevingsvariabelen - `DOMAIN` – domeinnaam voor Nginx/TLS (bijv. `mijn-darts-app.nl`) - `TLS_EMAIL` – e-mailadres voor Let's Encrypt - `ENABLE_TLS` – `1` om TLS in te schakelen, anders `0` - `TLS_MODE` – `http` of `dns` - `DNS_PROVIDER` – bij DNS mode: `cloudflare`, `digitalocean`, `route53` - `DNS_CREDENTIALS_FILE` – pad naar credentials bestand (indien nodig) - `ENABLE_ONTWIKKELTOOLS` – `true` of `false` (admin testtools tonen/verbergen) - `ENABLE_UPDATE_CHECK` – `true` of `false` (GitHub updatecontrole) - `GITHUB_REPOSITORY` – repo slug voor update checker, bijv. `owner/repo` ### Voorbeeld: productie-installatie met HTTPS ```bash DOMAIN=mijn-darts-app.nl \ TLS_EMAIL=admin@mijn-darts-app.nl \ ENABLE_TLS=1 \ TLS_MODE=http \ ENABLE_ONTWIKKELTOOLS=false \ ENABLE_UPDATE_CHECK=true \ GITHUB_REPOSITORY=ROosterloo1988/zomercompetitie \ bash ./scripts/install_ubuntu.sh ``` ## Updaten ```bash git pull bash ./scripts/install_ubuntu.sh sudo systemctl restart zomercompetitie sudo systemctl reload nginx ``` ## Data locatie Standaard databasepad in productie: `/var/lib/zomercompetitie/zomercompetitie.db` ## Licentie MIT License -
