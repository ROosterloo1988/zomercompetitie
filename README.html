# 🎯 Zomercompetitie Een open-source, mobielvriendelijke (PWA) FastAPI webapplicatie voor het beheren van een wekelijkse dartscompetitie. Gebouwd met een focus op gebruiksgemak voor de organisatie: snelle invoer van uitslagen vanaf een tablet, automatische poule-generatie, een volautomatisch TV-toernooischerm (inclusief DartConnect-integratie) én een robuuste architectuur die de drukte in een volle kroeg moeiteloos aankan. ## ✨ Features ### 🛡️ Systeem & Veiligheid (Nieuw) - **Beveiligd Beheerpaneel:** Toegang tot de admin-omgeving is afgeschermd met een veilig (bcrypt) wachtwoord en persistente sessie-cookies. - **Spitsuur-Proof (WAL Mode):** De database is geoptimaliseerd met SQLite Write-Ahead Logging. Hierdoor kunnen tientallen darters tegelijkertijd de app gebruiken en uitslagen opslaan zonder dat het systeem vastloopt. - **Geavanceerde PWA (Service Worker):** De app installeert zich als een native app op mobiel. CSS en iconen laden razendsnel uit het cachegeheugen, terwijl uitslagen altijd 'live' van de server worden gehaald. Valt het internet in de kroeg even weg? Dan toont de app netjes de laatst bekende standen in plaats van een error. ### 📱 Organisatie & Spelerservaring - **Interactief Dashboard:** Spelers kunnen vanaf de startpagina wedstrijden openklikken om direct te zien wie de 180'ers en hoge finishes (100+) heeft gegooid. - **Supersnelle Aanwezigheidsinvoer:** Aanwezigheid wordt direct op de achtergrond opgeslagen zonder pagina-refresh of scroll-jumps. - **Eerlijke Poules:** Automatische indeling met anti-herhaling op basis van historische ontmoetingen. - **Geavanceerde Knock-out:** Ondersteunt automatische 'vrijlotingen' (byes) bij afwijkende aantallen (zoals een 3-speler knock-out). - **Statistieken & Highlights:** Bulk-invoer voor uitslagen inclusief 180'ers, 100+ finishes en snelle legs (≤15 darters). - **Seizoensarchief:** Overall standen, legsaldo's en een compleet archief van voorgaande avonden. - **Realtime Multi-Device Sync:** Scores, aanwezigheid en spelerstatussen worden live gesynchroniseerd tussen alle open tablets, telefoons en TV-schermen via WebSockets. - **Live Score-invoer:** Terwijl een beheerder uitslagen invult, verschijnen wijzigingen direct op andere apparaten zonder pagina-refresh. - **Slimme Opslaan-UX:** Wijzigingen worden lokaal/live bijgehouden en pas definitief verwerkt na expliciet opslaan. - **Realtime Presence Sync:** Aanwezigheid van spelers wordt direct zichtbaar op andere apparaten zonder schokkerige pagina-reloads. ### 📺 Live TV Toernooischerm (`/?tv=1`) - **Ultrawide Dark Mode:** Een prachtig volautomatisch toernooischerm dat perfect schaalt van 1080p TV's tot brede Ultrawide monitoren, gestyled in de kleuren van de app. - **DartConnect Integratie:** Voeg eenvoudig live DartConnect wedstrijd-URL's toe via het Admin-paneel. Deze worden in realtime naast het competitieschema op de TV getoond. - **Slimme Wedstrijd-logica:** Openstaande wedstrijden uit verschillende poules worden automatisch om-en-om "geritst", zodat het toernooi op alle banen soepel doorloopt. - **Realtime TV-updates (WebSockets):** Het TV-scherm werkt volledig realtime. Nieuwe uitslagen, aanwezigheid en knock-out voortgang verschijnen direct zonder handmatige refresh. - **🏆 Huldigingsscherm:** Zodra een speelavond op `CLOSED` wordt gezet, transformeert de TV automatisch naar een uitslagenscherm met de winnaar van de avond én de actuele overall standenlijst. ### ⚙️ Systeembeheer - **Update Checker:** Ingebouwde notificatie in het admin-paneel zodra er een nieuwe release op GitHub staat. - **Geautomatiseerde Back-ups:** Het systeem maakt via een cronjob elke nacht veilige back-ups van de database. --- ## 🛠 Tech stack - **Backend:** Python 3.10+, FastAPI, SQLAlchemy, SQLite (WAL mode) - **Security:** bcrypt (wachtwoord-hashing), Starlette SessionMiddleware - **Frontend:** Jinja2 Templates, Vanilla JS/CSS (Responsive & Volledige PWA Service Worker) - **Deployment:** Nginx (Reverse Proxy) + systemd (via meegeleverde bash scripts) --- ## 🚀 Get Installatie op Ubuntu server ```bash mkdir workspace cd workspace sudo apt update && sudo apt install git -y && git clone https://github.com/ROosterloo1988/zomercompetitie.git ## 🚀 Installatie op Ubuntu server Gebruik het installatiescript vanuit de repo root voor een frisse installatie: let op gebruik de installatie variabelen. ```bash cd /workspace/zomercompetitie bash ./scripts/install_ubuntu.sh ### Belangrijkste omgevingsvariabelen - `DOMAIN` – domeinnaam voor Nginx/TLS (bijv. `mijn-darts-app.nl`) - `ADMIN_PASSWORD` – (Optioneel) Stel direct het beheerderswachtwoord in - `TLS_EMAIL` – e-mailadres voor Let's Encrypt - `ENABLE_TLS` – `1` om TLS in te schakelen, anders `0` - `TLS_MODE` – `http` of `dns` - `DNS_PROVIDER` – bij DNS mode: `cloudflare`, `digitalocean`, `route53` - `DNS_CREDENTIALS_FILE` – pad naar credentials bestand (indien nodig) - `ENABLE_ONTWIKKELTOOLS` – `true` of `false` (admin testtools tonen/verbergen) - `ENABLE_UPDATE_CHECK` – `true` of `false` (GitHub updatecontrole) - `GITHUB_REPOSITORY` – repo slug voor update checker, bijv. `owner/repo` ### Voorbeeld: productie-installatie met HTTPS ```bash DOMAIN=mijn-darts-app.nl \ TLS_EMAIL=admin@mijn-darts-app.nl \ ENABLE_TLS=1 \ TLS_MODE=http \ ENABLE_ONTWIKKELTOOLS=false \ ENABLE_UPDATE_CHECK=true \ GITHUB_REPOSITORY=ROosterloo1988/zomercompetitie \ bash ./scripts/install_ubuntu.sh ``` 🔄 Updaten Als je de applicatie al hebt draaien en je wilt de code bijwerken, gebruik dan het speciale update-script. Dit script synchroniseert de nieuwe bestanden, maar behoudt je database, je ingelogde sessies en je beheerderswachtwoord. ```bash git pull bash ./scripts/update_ubuntu.sh sudo systemctl restart zomercompetitie ``` 💾 Data locatie & Back-ups Standaard databasepad in productie: /var/lib/zomercompetitie/zomercompetitie.db Nachtelijke back-ups (laatste 14 dagen) worden opgeslagen in: /var/lib/zomercompetitie/backups/ 📜 Licentie MIT License - Vrij te gebruiken en aan te passen.
